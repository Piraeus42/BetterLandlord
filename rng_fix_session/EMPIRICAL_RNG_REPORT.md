## 任务 2：稀有度判定端到端经验分布

### 原版行为确认

#### Q1: 一次 roll 还是多次 roll？

原版代码 `Pop-up.tscn__1.gd:1347`:
```gdscript
var rand_num = rand_range(0, 1)
```
生成**一次** rand_range(0,1)，然后级联比较累积阈值。模组的 `_bh_c_rarity_randf()` 也返回一次 float。两者 roll 次数一致。

#### Q2: card_pool 是否排除已拥有物品？

原版代码 `Pop-up.tscn__1.gd:1253-1257`:
```gdscript
card_pool = rarity_database["items"].duplicate(true)
for i in Items.items:
    card_pool[i.rarity].erase(i.type)
for i in Items.recently_destroyed_items:
    card_pool[i.rarity].erase(i.type)
```
**是**，主分支排除已拥有和最近销毁的物品。ELSE 分支（行 1424）重建**未过滤**的池。

#### Q3: 有放回还是无放回？

原版在单次 add_cards 事件内**无放回**——选取后 erase 该项（行 1399, 1406, 1426, 1448）。

### 移植说明

移植用 Python 3.13，GDScript → Python 差异处理方式：
- GDScript 64-bit signed 包裹通过 `& 0xFFFFFFFFFFFFFFFF` + 有符号转换模拟（`if >= 0x8000000000000000: - 0x10000000000000000`）
- GDScript `>>` 算术右移：所有中间值 bit 63 为 0（MASK_63 保证），故与 Python `>>` 一致
- `x << ((-rot) & 31)` 溢出：默认用 64-bit 包裹模拟（实测与 pure-big-int 版本结果相同，说明包裹不影响最终 MASK_31 后的结果）
- `math.floor` 等价于 GDScript `floor()`
